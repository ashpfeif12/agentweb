› 01 · THE SPEC

agent.json

One file at your domain root. Tells agents what you do, what actions are available, how to authenticate, and what rules to follow. Like robots.txt — for agents.

{ "name": "Acme Fashion", "spec_version": "1.0", "description": "Premium retailer", "capabilities": [ { "name": "search_products", "type": "query" }, { "name": "place_order", "type": "action" } ], "brand_voice": { "tone": "warm, never pushy" }, "endpoints": { "mcp": "mcp.acme.com/sse" } }
› QUICK START

Generate your agent.json in one command.

Pick your industry. Get capabilities, brand voice, and policies pre-filled. Edit it. Ship it.

~/myproject
$ npx agentweb init --industry retail --output agent.json ✓ Created agent.json ✓ 5 capabilities for retail ✓ Brand voice template ✓ Policy placeholders ✓ Escalation triggers Next: npx agentweb validate agent.json npx agentweb score yoursite.com
› 03 · MCP GENERATOR

Turn any API into an MCP server.

Point at your OpenAPI spec. Get a complete, typed, deployable MCP server with auth, Dockerfile, and a matching agent.json.

agentweb generate
$ npx agentweb generate --from openapi.yaml [1/5] Parsing input... 5 tools found [2/5] Generating code... Done [3/5] Generating agent.json Done [4/5] Package files... Done [5/5] Writing output... Done Output: ./mcp-server/ src/server.ts MCP server with typed tools src/auth.ts Authentication handling agent.json Auto-generated manifest Dockerfile Ready to deploy
› 04 · MIDDLEWARE

Proxy any site. Agent-ready in seconds.

A managed MCP proxy that sits in front of any website. Content caching, brand voice enforcement, rate limiting, escalation detection, analytics — all built in.

› 05 · COMMERCE
