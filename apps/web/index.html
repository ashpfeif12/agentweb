The spec

agent.json

A single file at your domain root that tells AI agents what you do. Like robots.txt for agents. Four required fields.

Fields
nameREQYour brand name
spec_versionREQAlways "1.0"
descriptionREQWhat you do
capabilitiesREQWhat agents can do
actionsI/O schemas for each action
authenticationOAuth2, API key, bearer
policiesRate limits, data handling
brand_voiceTone, prohibited topics
endpointsMCP, REST, GraphQL URLs
humansEscalation paths
Example
{
  "name": "Acme Fashion",
  "spec_version": "1.0",
  "description": "Premium retailer, 2k+ products",
  "capabilities": [
    { "name": "search_products",
      "type": "query" },
    { "name": "place_order",
      "type": "action" }
  ],
  "brand_voice": {
    "tone": "warm, never pushy"
  },
  "endpoints": {
    "mcp": "https://mcp.acme.com/sse"
  }
}
